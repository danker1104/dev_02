# 냉장고 알리미 기획서

## 1. 프로젝트 개요
- 프로젝트명: 냉장고 알리미
- 목적: 식재료 망각으로 인한 폐기와 중복 구매를 줄이는 생활형 냉장고 관리 서비스

## 2. 문제 정의
- 식재료를 구매한 뒤 바쁜 일상으로 존재를 잊어 유통기한이 지나 폐기되는 일이 자주 발생한다.
- 외출 중 또는 마트에서 장을 볼 때 집에 있는 식재료를 정확히 기억하지 못해,
  - 이미 있는 식재료를 중복 구매하거나,
  - 있는 줄 알고 구매하지 않아 필요한 요리를 못 하는 상황이 발생한다.

## 3. 핵심 목표
- 유통기한 임박 식재료를 사전에 알려 폐기를 줄인다.
- 외출 중 재고 확인을 통해 불필요한 구매를 줄인다.
- MVP에서는 개인/단일 기기 사용성을 먼저 검증하고, 이후 가족 공유 경험으로 확장한다.

## 4. 타깃 사용자
- 1인 가구 및 가족 단위 사용자
- 장보기 전후 식재료 관리가 번거로운 사용자
- 냉장고 재고/유통기한을 자주 잊는 사용자

## 5. 핵심 기능
### 5.1 식재료 스마트 등록
- 카메라 촬영으로 식재료명/유통기한 자동 인식
- 인식 결과를 사용자에게 확인받고 즉시 수정 가능
- 자동 인식 실패 시 수동입력으로 등록 완료 가능

### 5.2 식재료 DB 저장
- 식재료 정보 텍스트 기반 저장
- 이미지 원본은 저장하지 않음
- 저장 항목(초기): 식재료명, 유통기한, 보관 구분(냉장/냉동/실온), 수량

### 5.3 유통기한 알림
- 유통기한 3일 전에 앱 푸시 1회 발송
- 알림 클릭 시 해당 식재료 상세/목록으로 이동

### 5.4 냉장고 검색
- 앱에서 식재료명 검색 시 보유 여부 확인
- 중복 구매 방지에 활용

### 5.5 중복 식재료 처리
- 동일 식재료 재등록 시 개별 항목 분리 대신 수량 증가 기본 처리

## 6. 정책 및 범위
### 6.1 로그인 정책
- 초기 버전은 로그인 없이 시작
- MVP 단계에서는 개인/단일 기기 사용을 기본으로 검증
- 가족 공유는 추후 선택 로그인 도입을 전제로 확장

### 6.2 식재료 범위
- 전체 식재료 범위(신선식품, 가공식품 등)를 대상으로 운영

### 6.3 데이터 정책
- 개인정보 최소화 원칙
- 텍스트만 저장, 이미지 원본 미저장

### 6.4 알림 채널
- 앱 푸시만 사용

### 6.5 재고 생명주기 정책
- 상태 정의: 보유 -> 임박 -> 위험 -> 소진 / 폐기
- 소진 처리: 사용자가 수량 차감(기본 1단위) 또는 전량 소진 처리
- 소진: 사용 완료된 상태로 기록하며 목록에서는 숨김 처리
- 폐기: 유통기한 경과 또는 사용자 폐기 처리로 기록
- 소진/폐기 모두 최근 이력에서 확인 가능

### 6.6 중복 및 동일성 정책
- 기본 동일성 기준: 식재료명 + 보관 구분 + 단위(개/봉/팩) + 용량이 같으면 동일 품목으로 판단
- 단위 또는 용량 정보가 없으면 신규 항목으로 분리 저장 후 사용자가 병합 여부를 선택
- 동일 품목 재등록 시 수량 증가
- 동일 품목이더라도 유통기한이 다르면 가장 이른 유통기한을 알림 기준값으로 사용

### 6.7 알림 예외 정책
- 기본 정책은 유통기한 3일 전 1회
- 등록 시점 기준 남은 기간이 3일 이하인 항목은 저장 직후 즉시 주의 배지 표시
- 푸시 권한 거부 시 앱 진입 시점에 임박/위험 배너와 권한 허용 유도 모달을 함께 제공
- 권한 미허용 상태에서는 대시보드 상단에 상시 경고 배너를 유지
- 위험 항목이 있는 경우 앱 실행 직후 우선 확인 카드(임박/위험 목록) 노출

### 6.8 빈 화면/온보딩 정책
- 최초 진입 시 빈 화면에서 카메라 등록과 수동입력 2가지 시작 동선을 동시에 제공
- 첫 등록 완료 전까지 대시보드 상단에 등록 유도 가이드 노출

### 6.9 데이터 이전 정책(비로그인 단계)
- 기기 변경/재설치 시 데이터가 초기화될 수 있음을 온보딩에서 고지
- 추후 선택 로그인 도입 시 기존 데이터 이전 기능 제공

## 7. 핵심 화면
### 7.1 메인 대시보드 (우리 집 냉장고)
- 냉장고 식재료 상태를 컬러 코딩으로 시각화
  - 신선: 초록
  - 임박: 노랑
  - 위험: 빨강
- 보관 카테고리 탭 분리
  - 냉장 / 냉동 / 실온

### 7.2 스마트 등록 화면 (카메라/스캔)
- 카메라 가이드라인 제공(식재료/유통기한 촬영 유도)
- 인식 후 자동 팝업되는 등록 확인 모달 제공
- 모달에서 식재료명/유통기한 즉시 수정 가능
- 수동입력 전환 버튼 제공

### 7.3 재고 관리 화면 (소진/수량 조정)
- 품목별 수량 증가/감소 버튼 제공
- 전량 소진 처리 버튼 제공
- 임박/위험 품목 우선 정렬 제공

### 7.4 빈 화면/알림 예외 안내
- 등록 데이터가 없을 때 시작 가이드 화면 제공
- 푸시 권한 미허용 시 임박 항목 배너 안내

## 8. 사용자 흐름 (MVP)
1. 사용자가 식재료를 구매하고 앱에서 카메라로 등록한다.
2. AI가 식재료명/유통기한을 인식해 자동 입력한다.
3. 사용자가 확인/수정 후 저장한다.
4. 시스템이 유통기한 3일 전에 앱 푸시 알림을 보낸다.
5. 사용자는 식재료 사용 후 수량을 차감하거나 전량 소진 처리한다.
6. 사용자는 외출 중 검색으로 재고 보유 여부를 확인한다.

## 9.1 예외 흐름 (MVP)
1. 자동 인식 실패 시 수동입력 화면으로 즉시 전환한다.
2. 남은 유통기한이 3일 이하인 항목은 저장 직후 임박/위험 상태로 즉시 표시한다.
3. 푸시 권한이 없으면 앱 진입 시 임박/위험 배너를 보여 대체 안내한다.

## 9. 기대 효과
- 유통기한 경과 전 알림으로 식재료 폐기 감소
- 장보기 중 재고 확인으로 중복 구매 감소
- 식비 절감 및 음식물 쓰레기 감소
- 초기에는 개인 사용자의 식재료 관리 효율 향상, 이후 가족 공유 확장으로 가구 단위 효율 향상

## 10. 타당성 검토 요약
- 문제-해결 적합성: 높음
- 초기 출시 가능성(MVP): 높음
- 핵심 리스크: 자동 인식 오차 및 등록 과정의 사용자 피로
- 대응 방향: 자동 인식 + 빠른 수동 보정 UX, 예외 상태(인식 실패/알림 권한 거부) 명확화

## 11. MVP 성공 지표 (초기)
- 등록 성공률: 등록 시도 대비 저장 완료 비율
- 등록 완료 시간: 등록 시작부터 저장 완료까지 평균 소요 시간
- 알림 후 재방문율: 알림 수신 사용자 중 24시간 내 앱 재방문 비율
- 검색 기능 사용률: 주간 활성 사용자 중 검색 기능 사용 비율
- 사용자 체감 폐기 감소율: 월별 설문 기반 자기보고 지표
- 소진 전환율: 등록된 식재료 중 소진 처리된 비율
- 폐기율: 등록된 식재료 중 폐기 처리된 비율

## 12. 향후 확장 방향
- 가족 공유 기능 고도화(선택 로그인 기반 동기화)
- 품목별 알림 시점 개인화
- 자주 등록하는 식재료 자동완성/추천 강화
